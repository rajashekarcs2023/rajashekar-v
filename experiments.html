Experiments

Smaller things I've built, tried, or figured out — usually while solving a problem that didn't have an obvious answer.

—
Browser automation primitives. Built domain-specific tools for credit card iframe filling, OTP interception, and date/time selection — as part of a multi-stage browser automation system. The insight: custom domain tools dramatically outperform pure LLM approaches for repetitive, high-stakes form interactions. The LLM is good at reasoning about what to do; bad at reliably clicking the right thing in a dynamic UI.
Browser-use CDP Automation
—
Small model fine-tuning on edge hardware. Fine-tuned Qwen-3 on AWS Trainium for real-time GitHub security alert classification. The lesson: narrow tasks + well-curated training data + a small model can match frontier models at a fraction of the inference cost. Paired it with an end-to-end pipeline — GitHub webhooks, SIEM, WebSockets, automated IAM key rotation.
Fine-tuning Qwen-3 AWS Trainium Security
—
Live event ordering agent. Stateful LangGraph agent with real-time Square Payments integration, serving 100+ attendees across an evening. The hard part wasn't the payments — it was keeping conversation context coherent as people rejoined mid-order or asked follow-up questions about items they'd ordered an hour ago.
LangGraph Square API Stateful agents
—
Hackathon support bot with live retrieval. ReAct-style orchestration agent for a 1,200-person hackathon — retrieving answers from a live Discord FAQ channel, Google Docs, and a static knowledge base in parallel, with smart escalation routing to human organizers via webhooks when confidence was low. The routing heuristic mattered more than the retrieval.
ReAct RAG Discord OpenAI tool-calling
—
29-agent film pipeline with failure recovery. The multi-agent film studio wasn't just about orchestration — it was about making 29 coordinated agents reliable. Black-video detection with retry logic, parallel scene production, FFmpeg assembly, and auto-generated character reference images to maintain visual consistency across scenes. Orchestration at this scale requires treating failure as the default, not the exception.
Multi-agent Google Veo 3.1 FFmpeg Reliability
